Claude for Odoo

Use Anthropic Claude as your Odoo 19 AI provider — agents, tool calling, RAG, and web search all work out of the box.

Why Claude?

Odoo 19 ships with OpenAI and Google Gemini. This module adds Anthropic Claude as a third provider. Same UI, same agents, same tools — just pick a Claude model from the dropdown.

Features

Feature Description
Full AI Agent Support Claude works with Odoo's agent system — topics, tools, system prompts, RAG sources
Tool Calling Claude can call Odoo server actions — search records, create leads, update data
Web Search Per-agent toggle to let Claude search the web for current information (Anthropic server-side)
RAG / Sources Attach knowledge articles, PDFs, and websites — Claude uses them as context
Structured Output JSON schema support for consistent, machine-readable responses
File Support Upload images, PDFs, and text files — Claude processes them natively

Setup in 3 Steps

1. Install

Install Pan AI Pro from the Apps menu. Requires the built-in AI module (Enterprise).

2. Configure

Go to Settings → AI, enable "Use your own Anthropic account", and paste your API key.

3. Use

Create or edit an AI Agent → select a Claude model from the dropdown. Done.

Web Search

Enable the Web Search toggle on any Claude agent to let it search the web for real-time information. Powered by Anthropic's native search capability — no external services to configure.

Location-aware results are provided automatically using your company's address settings.

How It Works

This module extends the standard Odoo 19 AI infrastructure. It registers Anthropic as a provider alongside OpenAI and Google — no custom AI framework, no separate configuration.

All existing Odoo AI features work automatically with Claude: Ask AI, text improvement, chatter AI, AI server actions, and more.

Requirements

  • Odoo 19 Enterprise with the AI module (ai_app) installed
  • Anthropic API key from console.anthropic.com
  • No additional Python packages required

Security

API key storage Standard Odoo ir.config_parameter — same as OpenAI/Google keys
Access control System administrator only (base.group_system)
Environment variable ODOO_AI_CLAUDE_TOKEN — optional override, no key in database
